一、Vue 基础

1. Vue的基本原理

  1. Vue基本原理
    Vue会给数据（data里的内容）加上“监听器”，一旦数据被读取或修改，它能立刻知道。每个组件都有个“观察者”，会盯着用到的数据。当数据变了，“观察者”就会让组件重新渲染，更新页面。
    （Vue3用Proxy，比Vue2的Object.defineProperty监听更全面，能直接监听到对象新增/删除属性等情况。）

2. 双向数据绑定的原理

简单说就是“数据变，页面跟着变；页面改，数据也跟着变”。

  • 先给所有数据加“监听器”（Observer），数据一变就发通知。
  • 解析模板（Compile），把数据显示到页面，同时给输入框等元素绑上事件，页面改动时能更新数据。
  • 用“订阅者”（Watcher）当中间桥梁，数据变了就告诉页面更新，页面改了就告诉数据同步。
    三者配合，实现双向绑定。

4. MVVM、MVC、MVP的区别

可以的，但可以再精简些，突出核心区别，面试时更清晰：

  • MVC：View（界面）、Model（数据）、Controller（控制器）。用户操作触发Controller，Controller改Model，Model通知View更新。View和Model直接有关联（观察者模式）。
  • MVP：View、Model、Presenter（中间层）。Presenter代替Controller，View和Model完全分离，全靠Presenter协调——改Model后，Presenter主动更新View，解耦更彻底。
  • MVVM：View、Model、ViewModel（视图模型）。ViewModel是核心，和Model双向绑定，和View自动同步。数据变了View自动更，用户改View数据也自动存Model，开发者不用手动操作界面，专注数据就行。

这样说逻辑更顺，重点突出，面试时能快速讲清差异，对方容易get到~

5. Computed 和 Watch 的区别

面试时可以这样说：

  • computed（计算属性）：
    用来根据其他数据计算出新值，会缓存结果——只有依赖的数据变了，才会重新计算。适合简单的、同步的计算场景，比如用“ firstName + lastName ”拼接出“ fullName ”。

  • watch（侦听器）：
    用来监听某个数据的变化，数据一变就执行操作，没有缓存。支持异步（比如调接口），适合需要做复杂处理的场景，比如监听输入的搜索关键词，变化时去请求后台数据。

    6. Computed 和 Methods 的区别

核心区别：computed是“算值”（依赖数据→结果），有缓存；watch是“监听变化做操作”，支持异步，无缓存。

computed和methods都能处理数据得到结果，但核心区别在是否缓存：

  • computed会缓存结果，只有依赖的数据变了才重新计算，调用时直接用缓存，更高效。
  • methods每次调用都会重新执行函数，不管依赖有没有变。

比如页面多次显示“总价”，用computed算一次就够了；而点击按钮才需要执行的操作，用methods更合适。

13. v-if和v-show的区别

v-if和v-show的核心区别在于渲染方式和使用场景：

  • 渲染方式：v-if是动态添加/删除DOM元素，初始条件为假时不渲染；v-show是通过CSS的display属性控制显示隐藏，DOM始终存在。
  • 性能侧重：v-if切换时消耗更高（涉及DOM操作），但初始渲染成本低；v-show初始渲染成本高，但切换更高效。
  • 使用场景：v-if适合条件少变的情况（如权限控制）；v-show适合需要频繁切换显隐的场景（如tab切换）。

16. data为什么是一个函数而不是对象

组件的data如果是对象，多个组件实例会共享这个对象，一个实例改了数据，其他实例也会跟着变，就乱了。

而用函数的话，每次创建组件实例时，函数都会返回一个新的data对象。这样每个实例的data都是独立的，各自维护自己的数据，不会相互干扰，保证了组件复用的独立性。

21. Vue 单页应用与多页应用的区别

面试时可以这样简洁说：

单页（SPA）和多页（MPA）应用，核心差异看这几点：

  • 结构：SPA 一个主页面 + 组件拼功能；MPA 多个独立完整页面。
  • 体验：SPA 切换快、局部更；MPA 切换慢，整页刷。
  • 资源：SPA 公共资源只加载一次；MPA 每个页重复加载 。
  • 场景：SPA 适合交互多、对 SEO 要求低的（如后台系统 ）；MPA 适合要 SEO 收录（如官网 ）。
  • 成本：SPA 前期难、后期好维护；MPA 前期易、后期改着烦 。

一句话总结：交互优先、功能复杂选 SPA；页面要被搜索到、功能独立用 MPA 。

28. 对 React 和 Vue 的理解，它们的异同

React 和 Vue 的异同

相似点

  • 功能分工：核心库专注核心功能，像路由、全局状态管理这些都交给专门库来处理。
  • 构建支持：都有构建工具，能快速生成符合最佳实践的项目模板。
  • 性能优化：都用虚拟 DOM 来提升页面重绘性能。
  • 数据传递：都有 props ，方便组件间传递数据。
  • 组件化：都鼓励把应用拆成一个个组件，提高复用性。

不同点

  • 数据流：Vue 天生支持双向数据绑定，React 坚持单向数据流。
  • 虚拟 DOM：Vue 能更精准跟踪组件依赖，计算差异更快，不用每次都重渲染整个组件树；React 状态变化时，默认所有子组件重渲染，得手动用 PureComponent 等方法优化 。
  • 组件模板：Vue 模板像 HTML ，加了些属性，容易上手；React 用 JSX ，类似 JavaScript 语法扩展，并且 React 中组件在 render 函数中可直接调用，Vue 引入组件后还得在 components 声明 。
  • 数据监听：Vue 靠劫持数据的 getter/setter 精准知道数据变化，性能好；React 默认对比引用判断数据变化，不优化的话可能会有很多不必要的重渲染。
  • 扩展方式：React 用高阶组件（HOC）扩展，Vue 用 mixins 扩展，因为 Vue 的 HTML 模板不利于高阶组件实现。
  • 构建工具：React 用 Create React APP ，Vue 用 vue - cli 。
  • 跨平台：React 有 React Native ，Vue 有 Weex 。

记 React 和 Vue 对比，可以抓核心差异，用“三记口诀”+ 关键特征串联：

一、相似点（记 3 个“都”）

  • 都分工明确（核心库 + 周边生态）
  • 都用虚拟 DOM（优化渲染性能）
  • 都组件化 + props 传值（拆模块、通数据）

二、不同点（记 5 个“最核心”）

  1. 数据流：Vue 双向绑定（数据能“来回”） | React 单向（数据“一条路”走到底）
  2. 模板写法：Vue 像 HTML（带指令） | React 用 JSX（和 JS 混写）
  3. 数据监听：Vue 自动劫持（数据变了自己知道） | React 默认对比引用（得手动优化）
  4. 扩展方式：Vue 靠 mixins（“混合”代码） | React 用高阶组件（“套娃”组件）
  5. 渲染优化：Vue 依赖追踪（只更需要的） | React 全量渲染（得手动拦着）

29. Vue的优点

  • 轻量：只关注视图层，体积小，就几十 kb 。
  • 好学：国人开发，中文文档，上手没难度。
  • 双向绑定：数据操作简单，保留了 angular 的优势。
  • 组件化：类似 react ，能封装和重用 HTML ，做单页面应用很厉害。
  • 结构分离：视图、数据、结构分开，改数据方便，不用动逻辑代码。
  • 虚拟 DOM：避免原生 DOM 操作的性能损耗，操作 DOM 更高效。
  • 速度快：和 React 比，操作虚拟 DOM 时性能更优。

三、Vue 优点（记 3 个“轻、简、快”）

  • 轻：体积小，只聚焦视图层
  • 简：中文生态好上手，双向绑定 + 组件化简单
  • 快：虚拟 DOM 优化 + 自动依赖追踪，渲染效率高

一句话总串：
React 和 Vue 都走“核心 + 生态”，靠虚拟 DOM 和组件化吃饭；但 Vue 更“亲民”（像 HTML、双向绑定），React 更“硬核”（JSX、单向流）。Vue 优点就是轻量、好懂、跑得快，适合快速落地；React 灵活度高，适合复杂场景折腾。

37. 对 SPA 单页面的理解，它的优缺点分别是什么？

SPA（单页应用）就是整个应用只有一个主页面，加载一次后，后续操作只换内容不刷新整个页面，靠路由切换显示不同内容。

优点：

  • 用户体验好，切换内容快，没有整页刷新的卡顿；
  • 服务器压力小，不用重复加载公共资源；
  • 前后端分工清楚，前端管交互，后端管数据。

缺点：

  • 第一次打开时要加载很多资源，可能慢；
  • 浏览器的前进后退不好用，得自己处理；
  • 对搜索引擎不友好，难被收录。

二、生命周期

1. 说一下Vue的生命周期

Vue的生命周期就是组件从创建到销毁的整个过程，分8个主要阶段，按顺序记关键节点和作用：

  1. beforeCreate：刚创建，数据和方法还没初始化，啥都拿不到。
  2. created：数据、方法初始化好了，但页面还没渲染（没DOM），适合做初始数据请求。
  3. beforeMount：模板编译好了，但还没挂载到页面上。
  4. mounted：页面渲染完成（有DOM了），可以操作DOM或发起后续请求。
  5. beforeUpdate：数据变了，但页面还没更新。
  6. updated：数据变了，页面也更新完了，能操作最新DOM。
  7. beforeDestroy：组件要销毁了，还能访问实例，适合清理定时器等。
  8. destroyed：组件彻底销毁，所有绑定、监听都没了。

另外，被 keep-alive 缓存的组件，切换时会触发 activated （激活）和 deactivated （停用）。

简单说就是：创建→挂载→更新→销毁，每个阶段分“前”“后”两步，记清楚每个阶段能做啥就行。

三、组件通信

1.组件通信的方式：

组件通信主要分几种场景，实习面试记核心用法就行，不用记代码：

  1. 父子组件通信
  • props / emit：最常用。父组件用props给子组件传数据；子组件用emit触发事件，给父组件传数据。
  • ref / refs：父组件给子组件加ref属性，通过this.refs.子组件名直接访问子组件的方法和数据。
  • $parent / children：子组件用this.parent 访问父组件，父组件用 this.$children（数组）访问子组件，但不太推荐，容易出问题。
  • provide / inject：父组件用 provide 提供数据，深层子组件用 inject 接收，适合隔代传值（非响应式）。
  1. 兄弟组件通信
  • eventBus（事件总线）：创建一个空的Vue实例当“桥梁”，一个组件用 $emit 发事件，另一个用 $on 监听，适合简单场景。
  1. 隔代/任意组件通信
  • $attrs / listeners：爷爷组件的数据/事件，爸爸组件不用处理，直接通过v-bind="attrs" 和 v-on="$listeners"传给孙子组件，避免层层传递。
  • Vuex/Pinia：复杂项目用，全局状态管理，所有组件都能读写共享数据（实习可能不深问，知道用途就行）。

总结：父子用 props/$emit 最基础，兄弟用 eventBus ，隔代用 $attrs 或 provide/inject ，复杂场景提一下Vuex就行，简单好记~
