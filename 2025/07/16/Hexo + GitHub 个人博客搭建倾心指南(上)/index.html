Hexo + GitHub 个人博客搭建倾心指南(上)

Created2025-07-16|Updated2025-07-16
|Post Views:
目标：搭建可自定义域名的静态博客，掌握前端工程化、Git 协作和部署流程。
技术栈：Node.js + Hexo + Git + GitHub Pages

第一部分

node.js的安装和环境配置
下载了一个nvm-desktop，是一个直观易用的Node.js版本管理桌面应用，可帮助开发者便捷地管理多个Node.js版本。有了这个工具，你再也不用去设置哪些繁琐的配置了。所有的配置、镜像源、安装位置等等等，都可以可视化操作。有手就行！！！
而且你可以为某个项目单独设置一个NodeJS版本，且不影响全局NodeJS版本和其他项目版本。比如你接手了一个古老的项目，他的Node版本很低，你就可以使用这个功能。

一.下载node js

1.可以通过node-v查看node版本，通过npm-v查看npm的版本，注意，如果权限不足的话，就是要以管理员身份运行终端（按下Win + R组合键，打开“运行”对话框 。在对话框中输入“cmd” ，然后同时按下Ctrl + Shift + Enter组合键 ，即可打开具有管理员权限的命令提示符。）
2.环境配置：

  • 设置npm全局模块安装路径：在非系统盘创建一个文件夹，比如 D:\nodejs_global ；打开命令提示符，执行 npm config set prefix “D:\nodejs_global” 。
  • 设置npm缓存路径 ：同样在非系统盘创建一个文件夹，比如 D:\nodejs_cache ；在命令提示符中执行 npm config set cache “D:\nodejs_cache” 。
  • 配置环境变量：右键点击“此电脑”，选择“属性”，进入“高级系统设置” ，点击“环境变量”。在系统变量的“Path”中，添加 D:\nodejs_global （前面设置的全局模块安装路径）。

二.安装git————“管好代码的历史，方便多人一起写代码”
1.安装git，Hexo 部署到 GitHub 需通过 Git 推送代码，需安装 Git 客户端。安装并配置Git ,git是一个并源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理
2.验证安装：打开 CMD，输入：git –version

三.生成SSH Keys
1.生成ssh ssh-keygen -t rsa -C “你的邮箱地址”
2.测试ssh是否绑定成功
Here ssh -T git@github.com

四.全局安装 Hexo CLI
Hexo 和 Hexo CLI 的区别
Hexo：
是一个基于 Node.js 的静态博客框架，本质是一套用于生成静态网页的程序代码（核心库）。它定义了博客的结构、主题、插件机制等功能，但本身不是一个可直接运行的命令。
Hexo CLI：是 Hexo 的指令集，命令行工具（Command Line Interface），简称 hexo-cli 。它是一个封装了 Hexo 核心功能的工具包，让你可以通过终端命令（如 hexo init 、 hexo g 、 hexo d ）快速操作 Hexo，而不需要手动编写复杂的 JavaScript 代码。
1.npm install hexo-cli -g
hexo-cli ：Hexo 的命令行工具包
2.验证安装是否成功
hexo -v

五.初始化 Hexo 项目（创建博客）
1、创建一个名为 Blog 的文件，在里面启用 Git Bash Here
Hexo init blog
2.初始化hexo
hexo init
3.生成页面
hexo g
4、生成本地的hexo页面
hexo s
5、访问打开本地服务区
http://localhost:4000/
6.本地文件上传到GitHub上面
hexo d

六.访问GitHub博客，创建GitHub仓库
1.用户名.github.io，这个就是你创建的新仓库
强制命名规则：
若想通过 https://你的用户名.github.io 这种默认域名访问博客，必须创建一个 与你的GitHub用户名完全一致 的仓库,你就可以通过这个网址访问博客了~

这里的“仓库名”是GitHub强制要求的 域名映射地址。你可以在博客内容中自定义标题、描述等，但访问地址始终由仓库名决定。

2.访问博客，开始的页面是初始化页面，没有做美化和增加内容。

第二部分

blog文件夹介绍：

_config.yml ：俗称站点配置文件，很多与博客网站的格式、内容相关的设置都需要在里面改。

node_modules :存储Hexo插件的文件，可以实现各种扩展功能。一般不需要管

scaffolds ：模板文件夹，里面的 post.md 文件可以设置每一篇博客的模板。具体用起来就知道 能干嘛了。

source ：非常重要。所有的个人文件都在里面！ 新建的博客中， source 文件夹下默认只有一个子文件夹—— _posts 。我们写的博客都放在这个子文件夹里面

themes ：主题文件夹，可以从Hexo主题官网或者网上大神的Github主页下载各种各样美观的主 题，让自己的网站变得逼格高端的关键！

这些 Hexo 指令是用来管理和部署你的博客的，具体作用如下：

  1. hexo cl
  • 作用：清除 Hexo 生成的缓存文件（ db.json ）和已生成的静态文件（ public 文件夹）。
  • 场景：当你更换主题、修改配置或内容后，发现网站更新不生效时，运行这个命令可以清除旧的缓存和文件，确保新的修改能够正确生成。
  1. hexo g
  • 作用：生成静态文件。 Hexo 会根据你的 Markdown 文章、主题模板等生成最终的 HTML、CSS、JavaScript 等静态文件，存放在 public 文件夹中。
  • 场景：每次修改文章或配置后，需要运行这个命令来生成新的静态文件，以便预览或部署。
  1. hexo s
  • 作用：启动本地服务器，用于在本地预览你的博客。
  • 场景：运行后，你可以在浏览器中访问 localhost:4000 来查看博客的实时效果，方便调试和预览。
  1. hexo d
  • 作用：部署到远程服务器（如 GitHub Pages）。
  • 场景：当你在本地预览确认无误后，运行这个命令可以将生成的静态文件推送到远程服务器，使你的博客可以在互联网上访问（如 yourname.github.io ）。

总结

  • 本地开发流程：修改内容 → hexo cl （清除旧文件） → hexo g （生成新文件） → hexo s （本地预览）。
  • 部署到线上：修改内容 → hexo cl → hexo g → hexo d （部署到 GitHub）。
