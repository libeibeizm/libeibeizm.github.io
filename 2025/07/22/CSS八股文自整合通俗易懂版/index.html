3. display的属性值及其作用

属性值 作用
none 元素不显示，并且会从文档流中移除。
block 块类型。默认宽度为父元素宽度，可设置宽高，换行显示。
inline 行内元素类型。默认宽度为内容宽度，不可设置宽高，同行显示。

4. display的block、inline和inline-block的区别

display：规定元素应该生成的框的类型

（1）block（块类型）： 会独占一行，多个元素会另起一行，可以设置width、height、margin和padding属性；

（2）inline（行内元素类）： 元素不会独占一行，设置width、height属性无效。但可以设置水平方向的margin和padding属性，不能设置垂直方向的padding和margin；

（3）inline-block： 将对象设置为inline对象，但对象的内容作为block对象呈现，之后的内联对象会被排列在同一行内。

对于行内元素和块级元素，其特点如下：

（1）行内元素

  • 设置宽高无效；
  • 可以设置水平方向的margin和padding属性，不能设置垂直方向的padding和margin，只能被元素内部内容撑开，所以如果有文字图片在一起时，图片会比较大，想实现文字环绕图片效果，就需要用到浮动（后来的普通元素就会装作浮动元素不存在的样子，布局到浮动元素下面，浮动元素会遮挡住普通元素，不过普通文档里面的文本节点会识别到浮动元素，就会绕开他，形成文字环绕图片。）；
  • 不会自动换行，多个元素可以占一行，挤不下时，新换一行继续挤；

（2）块级元素

  • 可以设置宽高，独占一行；
  • 设置margin和padding都有效；
  • 可以自动换行；
  • 多个块状，默认排列从上到下。

隐藏元素的方法有哪些

  • display: none：渲染树不会包含该渲染对象，因此该元素不会在页面中占据位置，也不会响应绑定的监听事件。
  • visibility: hidden：元素在页面中仍占据空间，但是不会响应绑定的监听事件。
  • opacity: 0：将元素的透明度设置为 0，以此来实现元素的隐藏。元素在页面中仍然占据空间，并且能够响应元素绑定的监听事件。
  • position: absolute：通过使用绝对定位将元素移除可视区域内，以此来实现元素的隐藏。

11. 对盒模型的理解

CSS3中的盒模型有以下两种：标准盒子模型、IE盒子模型 盒模型都是由四个部分组成的，分别是margin、border、padding和content。

标准盒模型和IE盒模型的区别在于设置width和height时，所对应的范围不同：

  • 标准盒模型的width和height属性的范围只包含了content，
  • IE盒模型的width和height属性的范围包含了border、padding和content。

可以通过修改元素的box-sizing属性来改变元素的盒模型：

  • box-sizeing: content-box表示标准盒模型（默认值）
  • box-sizeing: border-box表示IE盒模型（怪异盒模型）

6. link和@import的区别

两者都是外部引用CSS的方式，它们的区别如下：

  • link是XHTML标签，除了加载CSS外，还可以定义RSS等其他事务；@import属于CSS范畴，只能加载CSS。
  • link引用CSS时，在页面载入时同时加载；@import需要页面网页完全载入以后加载。
  • link是XHTML标签，无兼容问题；@import是在CSS2.1提出的，低版本的浏览器不支持。
  • link支持使用Javascript控制DOM去改变样式；而@import不支持。

16. 常见的图片格式及使用场景

给你通俗讲讲常见图片格式咋回事：

  1. BMP：基本没压缩，画质无损但文件超大，就像没挤水分的大包裹，占地方，适合对画质要求极高、不咋在意文件大小的场景 。
  2. GIF：用了压缩，文件小，还能做动画、搞透明效果，不过色彩种类少。像简单动画、小表情包，用它就挺合适，文件小好传播 。
  3. JPEG：压缩会损失点画质，但色彩丰富，存照片常用它，拍的生活照、风景照存成这格式。但要是存企业 logo 这类线条、色块简单的，一压缩容易糊，还别用它 。
  4. PNG - 8：能替代 GIF，文件比 GIF 还小点儿，也支持透明，要是不用动画，选它比 GIF 好，做些简单图标、小插画，用这个 。
  5. PNG - 24：画质无损，色彩也全，文件比 BMP 小很多，不过比前面几种格式大。要是图片色彩多、还得高质量，又不想文件大到离谱，用它 。
  6. SVG：是矢量图，放大也不会模糊，由线条、曲线组成，画 logo、小图标这些，放大缩小都清晰，就靠它

    二、页面布局

1. 常见的CSS布局单位

  • px（像素）：最基础的单位，是屏幕显示的最小区域。分两种，CSS像素是开发者用的抽象单位，物理像素是设备屏幕实际的最小点，固定不变。
  • %（百分比）：能让元素大小随浏览器或父元素变化，实现响应式。一般相对于直接父元素的尺寸。
  • em：相对长度，大小是父元素字体大小的倍数。如果父元素没设字体大小，就默认按浏览器的16px算。
  • rem：相对根元素（html标签）的字体大小，是CSS3新增的。方便做响应式，改根元素字体大小，所有用rem的元素都会跟着变。
  • vw/vh：和浏览器窗口（视图）有关。vw是窗口宽度的百分比（1vw=1%宽度），vh是窗口高度的百分比。还有vmin（取vw和vh里小的那个）、vmax（取大的那个）。

2. px、em、rem的区别及使用场景

  • px是固定的像素，设置后不会随页面大小变，适合适配设备少、分辨率影响小的情况。
  • em和rem更灵活，是相对单位，适合响应式布局。em看父元素，rem看根元素。
  • 要适配iPhone和iPad这种分辨率差别大的设备，用rem更方便。

4. 三栏布局的实现

三栏布局一般指的是页面中一共有三栏，左右两栏宽度固定，中间自适应的布局，三栏布局的具体实现：

  • 利用绝对定位，左右两栏设置为绝对定位，中间设置对应方向大小的margin的值。

  • 利用Flex布局（最推荐，简单灵活）

  • Flex 容器能干嘛（核心能力）：白话：父元素变成“Flex 容器”，左右栏占固定宽度，中间栏说“剩下的我全包”，自动撑满。

    • 当一个元素设为 display: flex ，它就变成 Flex 容器，里面的子元素（Flex 项目）能：

    1. 灵活排列方向：子元素可以横着排（默认）、竖着排，甚至reverse反向排，比如做导航栏、垂直列表。

    2. 自动分配空间：子元素能“平分”或“按比例分”容器剩余空间，轻松实现自适应布局（比如两栏/三栏布局，中间栏自动撑满）。

    3. 控制对齐方式：子元素能整体居左、居中、居右，或上下对齐，解决垂直居中、两端分布等常见需求。

  • 利用浮动+margin，左右两栏设置固定大小，并设置对应方向的浮动。中间一栏设置左右两个方向的margin值，注意这种方式**，中间一栏必须放到最后：**
  • 圣杯布局，利用浮动和负边距来实现。父级元素设置左右的 padding，三列均设置向左浮动，中间一列放在最前面，宽度设置为父级元素的宽度，因此后面两列都被挤到了下一行，通过设置 margin 负值将其移动到上一行，再利用相对定位，定位到两边。
  • 双飞翼布局，双飞翼布局相对于圣杯布局来说，左右位置的保留是通过中间列的 margin 值来实现的，而不是通过父元素的 padding 来实现的。本质上来说，也是通过浮动和外边距负值来实现的。

7. 对Flex布局的理解及其使用场景

一、基本概念

  • 啥是Flex布局：把容器（比如一个div）的 display 设为 flex ，它就变成“弹性容器”，里面的子元素自动成“弹性项目” 。一旦设了Flex，子元素的 float 、 clear 、 vertical-align 这些属性就没用了。

二、容器的两根轴

容器默认有 主轴（水平方向，项目默认沿这排）和 交叉轴（垂直方向，和主轴垂直 ）。

三、容器上的6个属性（管整体排列规则）

  1. flex-direction：决定主轴方向，比如项目是从左到右排，还是从上到下排。
  2. flex-wrap：一行排不下时咋换行，比如挤一挤不换行，还是自动换到下一行。
  3. flex-flow：上面 flex-direction 和 flex-wrap 的简写，省得写俩属性。
  4. justify-content：控制项目在 主轴 上咋对齐，比如靠左、靠右、居中，或者均匀分布。
  5. align-items：控制项目在 交叉轴 上咋对齐，比如垂直居中、顶端对齐啥的。
  6. align-content：有多行项目时，控制这些行在交叉轴上咋对齐（只有一行时没用 ）。

四、项目上的6个属性（管单个项目咋排）

  1. order：改项目的排列顺序，数字越小越靠前。
  2. flex-grow：有多余空间时，项目咋放大。默认0（不放大 ），设成1就能跟着分剩余空间。
  3. flex-shrink：空间不够时，项目咋缩小。默认1（会缩小 ），设0就不缩小。
  4. flex-basis：分配空间前，项目占主轴多大地方，默认是自身大小。
  5. flex：上面 flex-grow 、 flex-shrink 、 flex-basis 的简写，常用写法比如 flex: 1 ，快速设置放大缩小规则。
  6. align-self：单个项目想搞特殊对齐，能覆盖容器的 align-items 设置。

五、简单总结

Flex布局就是让容器里的项目，能通过主轴、交叉轴的各种属性，灵活控制排列方式。，特别适合做响应式布局（不同设备屏幕下，布局自动适配 ），现在网页开发经常用它搞定复杂排版～

三、定位与浮动

1. 为什么需要清除浮动？清除浮动的方式

咱通俗讲讲“清除浮动”这事儿，就像收拾房间，浮动元素乱跑影响布局，得“清理” ：

一、为啥要清除浮动？（先懂浮动啥问题）

  • 浮动的“坑”：元素设了 float （浮动），就会“脱离标准流”。加了 float: left/right ，元素就“飘”到父容器左边/右边，不再占标准流的位置 。比如一排图片，给它们设 float: left ，就会挨个往左排。 父容器如果没设高度，就没法被浮动的子元素撑起来，就会有“高度塌陷” 。
  • 浮动后的元素：正常情况下，只能在父元素内浮动，其布局既像块级元素可以设置宽高，又像行内元素可以多个元素在一行。
  • 带来的麻烦：
  • 父容器高度没了，下面的元素会“往上跑”，和浮动元素重叠，布局乱套；
  • 同行的非浮动元素，也会被浮动元素挤到奇怪位置，页面结构全乱。

二、清除浮动的常用办法（怎么“收拾”）

  1. 给父元素定高度：比如父容器是 div ，直接写 height: 200px 。父容器有了高度，就不会因子元素浮动“塌陷”，简单直接，但得提前知道子元素多高，不灵活。
  2. 加空标签清浮动：在最后一个浮动元素后面，加个空的 div ，给它设 clear: both 。这个空标签就像“墙”，拦住浮动，让父容器能被撑起来。但会多写无用标签，不推荐。
  3. 父元素用 overflow ：给父容器加 overflow: hidden 或者 overflow: auto 。浏览器会自动计算父容器高度，把浮动元素“包裹”住，简单好用，就是内容多了可能出现滚动条（ overflow: auto 会智能判断）。

简单说， clear 属性清除浮动的原理和用法是这样的：

一、 clear 是干啥的？

  • 不是真把浮动“删掉”，而是让 当前元素自己的边，别和“前面的浮动元素”贴在一起 。 比如前面有个左浮动的元素，当前元素设 clear: left ，那当前元素就会跑到前面左浮动元素的下边，不挨着它；设 clear: both ，就是不管前面是左还是右浮动，当前元素都躲到它们下边 。

二、为啥不用 left / right ，直接用 both ？

  • 因为浮动元素要么左、要么右， clear: left 生效时，右边就算有浮动也管不着（ clear 只看“前面的”浮动 ），这时候 left 效果其实和 both 差不多；同理 clear: right 生效时也类似 both 。 所以直接用 clear: both 更省事，不用纠结左还是右～

三、伪元素清除浮动咋回事？

  • 伪元素（比如 ::after ）默认是“行内元素”，而 clear 只有对 块级元素 才有效 。所以用伪元素清浮动时，得给它加 display: block （或 table 等块级表现的属性 ），再写 clear: both 。这样伪元素就像一个“看不见的块级元素”，把浮动的影响拦住，让父容器能正常撑起高度，布局不乱～

  • 总结一下： clear 是让元素躲开发前面的浮动，常用 both ；伪元素清浮动，得把伪元素改成块级，再用 clear: both 兜底～

    3. 对BFC的理解，如何创建BFC

一、BFC 是啥

BFC（Block Formatting Context，块格式化上下文 ），简单说就是一个独立的“小天地”（容器），里面的元素按照自己的规则布局，绝对不会影响到外面的元素布局（原则）。

二、咋创建 BFC（满足任一条件即可）

  1. 整个页面的根元素（ body 标签，天生就是 ）。
  2. 元素设浮动（ float: left/right 等，不能为 none ）。
  3. 元素设绝对/固定定位（ position: absolute/fixed ）。
  4. 改显示方式（ display: inline - block / table - cell / flex 等 ）。
  5. 溢出控制（ overflow: hidden/auto/scroll ）。

三、BFC 能干啥（实用作用）

  1. 解决 margin （外边距） 重叠：比如两个盒子上下 margin 挨一起会“合并”（因为兄弟元素之间的外边距在垂直方向会取最大值而不是取和），把它们放进各自 BFC（比如外层套个设了 overflow: hidden 的盒子 ），就不重叠啦。（当然也可以一个设为margin-bottom：20，另一个设置为padding-top:20）
  2. 解决浮动元素令父元素高度塌陷问题：父盒子里的子元素浮动后，父盒子高度会“塌”成 0 。给父盒子设 overflow: hidden （让父盒子变成 BFC ），父盒子为了保正BFC的原则，不得不包住浮动的子元素，高度正常显示。
  3. 做自适应变化两栏布局：左边盒子 float: left 定宽，右边盒子设 overflow: hidden 触发 BFC ，右边就会自动占满剩下空间，还不跟左边重叠，实现“左边固定、右边自适应” 。

6. position的属性有哪些，区别是什么

position有以下属性值：

属性值 概述
absolute 生成绝对定位的元素，相对于static定位以外的一个父元素进行定位。元素的位置通过left、top、right、bottom属性进行规定。
relative 生成相对定位的元素，相对于其原来的位置进行定位。元素的位置通过left、top、right、bottom属性进行规定。
fixed 生成绝对定位的元素，指定元素相对于屏幕视⼝（viewport）的位置来指定元素位置。元素的位置在屏幕滚动时不会改变，⽐如回到顶部的按钮⼀般都是⽤此定位⽅式。
static 默认值，没有定位，元素出现在正常的文档流中，会忽略 top, bottom, left, right 或者 z-index 声明，块级元素从上往下纵向排布，⾏级元素从左向右排列。
inherit 规定从父元素继承position属性的值

前面三者的定位方式如下：

  • relative： 元素的定位永远是相对于元素自身位置的，和其他元素没关系，也不会影响其他元素。

  • fixed： 元素的定位是相对于 window （或者 iframe）边界的，和其他元素没有关系。但是它具有破坏性，会导致其他元素位置的变化。

  • absolute： 元素的定位相对于前两者要复杂许多。如果为 absolute 设置了 top、left，浏览器会根据什么去确定它的纵向和横向的偏移量呢？答案是浏览器会递归查找该元素的所有父元素，如果找到一个设置了position:relative/absolute/fixed的元素，就以该元素为基准定位，如果没找到，就以浏览器边界定位。

7. display、float、position的关系

这三个属性在网页布局中紧密相关，它们的相互作用决定了元素的显示效果，具体关系可看作一种优先级机制：

  • display为none时：元素不会显示，也不占据页面空间，此时position和float属性对元素没有任何影响。比如一个设置了 display: none 的按钮，不管它的position和float怎么设置，都不会在页面上出现。

  • position为absolute或fixed时：float属性会失效，display的值会被强制转换为table或block（具体取决于初始值）。像一个设置了 position: absolute 的导航栏，它会脱离文档流，不再受float属性控制，并且会以类似块级元素的方式显示。

  • position不是absolute或fixed，float不为none时：display的值会按照特定规则转换。如果position是relative且float存在，元素会先根据float规则定位，然后relative再相对于浮动后的最终位置进行定位。例如一个列表项设置了 float: left 和 position: relative ，它会先向左浮动，然后relative的偏移是基于浮动后的位置。

  • float为none时：如果是根元素，display属性会按规则转换；如果不是根元素，则display保持原有设置。

    总的来说，可以把它看作是一个类似优先级的机制，”position:absolute”和”position:fixed”优先级最高，有它存在的时候，浮动不起作用，’display’的值也需要调整；其次，元素的’float’特性的值不是”none”的时候或者它是根元素的时候，调整’display’的值；最后，非根元素，并且非浮动元素，并且非绝对定位的元素，’display’特性值同设置值。

8. absolute与fixed共同点与不同点

absolute和fixed是两种定位方式，它们既有相似之处，也有明显区别：

共同点：

 * 两者都能改变行内元素的呈现方式，让其类似 display: inline-block ，可以设置宽高；

  • 都会使元素脱离普通文档流，不占据文档的物理空间，其他元素的布局就像它们不存在一样；

  • 并且都能覆盖非定位的文档元素。比如一个用 absolute 或 fixed 定位的广告弹窗，会浮在页面其他内容之上。

    不同点：

    • abuselute与fixed的根元素不同，abuselute的根元素可以设置，fixed根元素是浏览器。
    • 在有滚动条的页面中，absolute会跟着父元素进行移动，fixed固定在页面的具体位置。
