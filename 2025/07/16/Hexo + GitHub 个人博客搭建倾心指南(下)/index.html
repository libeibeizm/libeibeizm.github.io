第三部分

更换主题（Butterfly 示例）

Butterfly 主题安装：git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly

修改 Hexo 根目录下的 _config.yml，把主题改为 butterfly：theme: butterfly

如果你没有 pug 以及 stylus 的渲染器，请下载安装：npm install hexo-renderer-pug hexo-renderer-stylus –save

为了减少升级主题后带来的不便，请使用以下方法：在 hexo 的根目录创建一个文件 _config.butterfly.yml，并把主题目录的 _config.yml 内容复制到 _config.butterfly.yml 去，以后只需要在 _config.butterfly.yml 进行配置就行。

修改主题配置文件 _config.butterfly.yml

#footer是否显示图片背景(与 top_img一致) footer_bg: true

访问人数busuanzi:
site_uv: true
site_pv: true
page_pv: true

如何撰写博客

1.直接找到这个地方，然后添加一个.md

2.用typora打开，即可编辑，记得保存。

3.在VS Code中编辑Hexo博客文章时，在文章开头Front - Matter区域（以 — 包裹 ）通过 title 字段设置标题

4.执行 npx hexo g :hexo generate 的简写命令 在Hexo博客系统中的作用是生成静态网页。再执行 npx hexo d 命令启动网站服务器，然后在浏览器中访http://localhost:4000 来查看文章效果。

第四部分

面试常见问题

1.项目难点？

  - 解决部署时的 YAML 格式敏感问题（空格/缩进）

  - 处理 GitHub Pages 的 HTTPS 强制跳转与自定义域名兼容性

2.为何选择 Hexo？

  - 静态博客速度快、安全性高

  - 丰富的主题/插件生态（如 Butterfly 主题）

  - 无缝集成 GitHub Pages 实现自动化部署

3.Git 协作流程？

  - 代码修改 → git add . → git commit -m "message" → git push origin main

  - 部署命令 hexo d 自动触发 GitHub Pages 构建

4.如何优化博客性能？

  - 使用 CDN 加速静态资源

  - 压缩 HTML/CSS/JS（通过 Hexo 插件）

  - 懒加载图片

5.自定义域名原理？

  - CNAME 文件指定域名 → DNS 解析将域名指向 GitHub 服务器 IP → GitHub 验证所有权并托管内容

! 提示：1.实操中遇到问题时，优先查阅 [Hexo 官方文档](https://hexo.io/zh-cn/docs/) 和 GitHub 仓库 Issues。

2.在Git Bash里，若想在一行执行多条命令，命令间应用分号’’;’’隔开。

3.在Visual Studio Code中搜索“xxx”的方法

(1).快捷键方式：按下Ctrl + F会在编辑器界面上方出现搜索框，在搜索框中输入“xxx”编辑器会自动高亮显示中所有出现“xxx”的位置。

(2).菜单方式：点击菜单栏中的“编辑”，选择“查找”，同样会弹出搜索框，输入“xxx”进行搜索。

4.hexo更换背景图片 背景图片参考网址： https://wallhaven.cc/ ，https://wall.alphacoders.com/， https://bz.zzzmh.cn/index

注：本方法解决的是多次同步到GitHub上背景图片未成功的情况： npm i hexo-deployer-git hexo new post “新建博客文章名”， hexo cl && hexo g && hexo s直接更改原文件， 图片所在目录： hexo/themes/landscape/source/css/images/ ，图片名称： banner.jpg
