《一文看懂Node.js、nvm、npm、npx：谁是大厨？谁是调度员？》

Created2025-07-22|Updated2025-07-22
|Post Views:

如下：

  • node.js:首先刚开始的时候JavaScript只能在浏览器中运行，后来为了解决这个问题呢，就出现了node js，它可以充当一个，让JavaScript走出浏览器的户外炉灶，让它可以不仅在浏览器里运行，也可以在服务器端运行(就是后端)

  • nvm:随着node js的使用，又出现了非常多的版本，那作为程序员来说的话，各个版本都有可能要使用到，所以就出现了NVM，它就充当了node.js版本的一个调度员哦，你使用什么它就能给你切换出来

  • npm:在项目开发当中需要用到包，包就相当于项目中的某一个功能模板，一个项目中就会涉及很多个包，那这些包其实有的，比如说像登录的这样一个功能的，其实很多项目都会用到这样的一个包那为了方便，就用到了NPM，它就是一个带我们安装不同功能包的一个安装器，你后面可以利用它嗯，一键升级，一键卸载，还有一键管理都OK，相当于你把工具带回家。

  • npx:那如果你只是想在你的项目里临时跑一个命令，而不需要添加整个包的时候，我们这个时候就会涉及到npx，跑完这个命令之后呃，它就离开了。相当于一个临时的工具用完就撤了。
